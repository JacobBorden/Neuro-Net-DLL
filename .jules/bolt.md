## 2024-05-24 - [Matrix Multiplication Cache Optimization]
**Learning:** Due to the row-major memory layout of the custom `Matrix<T>` class, the original `i-k-j` matrix multiplication logic using `sum += m_Data[i][j] * b.m_Data[j][k]` suffered from severe cache thrashing because the inner loop accessed non-sequential memory addresses in the right-hand operand. Applying loop interchange to iterate in `i-k-j` order with `c.m_Data[i][j] += a_ik * b.m_Data[k][j]` keeps memory access sequential for both `b` and `c`, yielding a significant cache performance boost without the overhead of manually transposing `b`.
**Action:** Always verify loop nesting order corresponds to the underlying contiguous memory layout (row-major vs column-major) for expensive N^3 matrix operations to maximize cache line utilization.
