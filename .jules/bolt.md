## 2024-05-24 - [Optimize Matrix Multiplication Loop Order]
**Learning:** Found a major performance bottleneck specific to this codebase's architecture: Matrix multiplication (`src/math/matrix.h`) was using `i-k-j` loop ordering. Because the custom `Matrix` implementation is row-major, this resulted in non-sequential memory access during the inner loop when modifying the result matrix `c.m_Data[i][k]` and reading `b.m_Data[j][k]`. Switching to `i-j-k` ensures sequential memory access for both read and write operations in the innermost loop. The matrix multiplication is now significantly faster. For 500x500 matrices, the benchmark duration fell from 13.7 seconds to 0.19 seconds.
**Action:** Always prefer `i-j-k` loop order for matrix multiplication in row-major implementations to take advantage of cache locality and avoid thrashing. Always build in `Release` mode before attempting to draw conclusions from benchmarks.
