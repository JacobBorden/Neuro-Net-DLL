## 2024-06-19 - Cache-friendly Loop Reordering in Custom Matrix Multiplication
**Learning:** The custom `Matrix<T>` class uses a row-major memory layout implemented as an array of `MatrixRow` pointers. The original OpenMP parallelized matrix multiplication used an `i-k-j` loop order which was causing severe cache misses. Specifically, the innermost loop (`j`) incremented the row index of the second matrix `B` (`b.m_Data[j][k]`), resulting in large memory jumps (non-sequential access) during each iteration.
**Action:** Always verify loop ordering against the underlying memory layout (row-major vs column-major) for multidimensional arrays. Reordering the loops to `i-j-k` ensures the innermost loop accesses elements sequentially in memory (`b.m_Data[j][k]` and `c.m_Data[i][k]` where `k` varies), which is highly cache-friendly and yields immediate speedups, even outweighing the overhead of zero-initializing the output matrix beforehand.
