## 2024-05-18 - [Optimize matrix multiplication cache locality]
**Learning:** Reordering the nested loops in matrix multiplication from standard `i-k-j` (where `k` is the dot product dimension inside) to an outer `i`, then `k`, then an inner `j` loop significantly improves cache locality for row-major matrices. Furthermore, using raw pointers `T* c_row = &c.m_Data[i][0]` inside the loops avoids overhead from `operator[]` lookups, providing massive speedups (~583ms down to ~60ms for 1000x1000 floats).
**Action:** Always structure nested loops to iterate sequentially through memory based on the underlying memory layout (row-major vs column-major). Utilize raw pointers in tight inner loops instead of repeated `operator[]` accesses when compiler auto-inlining/optimization isn't enough.
